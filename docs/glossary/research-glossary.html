Research Glossary

This glossary standardizes the working language for Coherascent Labs research so notes, experiments, benchmarks, and papers use the same operational definitions. The emphasis is on measurable meaning rather than rhetorical framing.

Last updated: April 2, 2026
Scope: logic, optimization, evaluation, neuro-symbolic systems
Usage Rules Research Language Logic Systems Optimization Evaluation Formalization

Usage Rules

The glossary is intended to function as a contract across the repository. If a future note, notebook, or paper uses a narrower definition, that deviation should be stated explicitly rather than assumed.

Purpose

Operational by Default

Terms should be interpreted in a way that supports experiment design, failure analysis, or benchmark construction. Ambiguous terms are acceptable internally only when their ambiguity is named.

Working Rules
  • Use glossary definitions by default across notes, experiments, and roadmap updates.
  • Prefer measurable definitions over aspirational language.
  • Revise terminology when experiments sharpen the meaning of a term.
  • Do not promote internal shorthand into public writing without formalization.

Research Language

System Vocabulary

Agents, Inference, Grounding

Agent
A system that performs multi-step inference or action selection under a specified objective. It is not assumed to be truthful or symbolic by default.
Probabilistic Agent
A model whose outputs depend on learned distributions, stochastic sampling, or uncertain internal representations. Large language models usually fall into this category.
Inference
The process by which a system derives outputs or state updates from inputs, priors, and learned or formal structure. It may be neural, symbolic, approximate, hybrid, deterministic, or probabilistic.
Grounding
Tying outputs to external constraints, verified data, formal systems, or interpretable state structure so claims are not merely statistically plausible.
Truth Properties

Faithfulness, Truth, Coherence

Hallucination
An output that presents unsupported or fabricated content as valid. It should always be judged against a task definition or evidence source, not vague intuition.
Faithfulness
The degree to which an output is actually supported by the system's reasoning process, evidence, or governing constraints rather than post-hoc explanation.
Truth Alignment
The degree to which a system is guided toward externally correct, internally consistent, and formally admissible outputs. It is not equivalent to user agreement or stylistic fluency.
Mathematical Coherence
Consistency between system behavior and the logical, optimization, and interface structures used to describe or constrain it.
Program Scope

Neuro-Symbolic Framing

Neuro-Symbolic AI
The integration of learned statistical models with explicit rules, symbolic structures, proof mechanisms, or verification modules.
Architectural Determinism
The degree to which system behavior is constrained by explicit and inspectable rules. It does not require identical outputs on every run, but it does require formally limited admissible behavior.
Control Layer
A mechanism placed around, above, or within a probabilistic model to enforce or encourage compliance with declared constraints.
Symbolic Module
A subsystem that manipulates explicit human-interpretable structures such as rules, clauses, variables, or proofs.

Logic and Constraint Systems

Classical Foundations

Logic Substrate

Boolean Satisfiability (SAT)
The problem of deciding whether there exists a Boolean assignment that makes a logical formula true. It provides a formal substrate for consistency and feasibility.
CNF
Conjunctive normal form, where a formula is written as an AND of clauses and each clause is an OR of literals. It is central for SAT representations and machine-checkable reasoning tasks.
DPLL
The Davis-Putnam-Logemann-Loveland procedure for SAT, built around branching, unit propagation, simplification, and backtracking.
Formal Logic
A rule-governed symbolic system for representing propositions, inference procedures, and validity relations relevant to verification and reasoning.
Constraint Handling

Rules, Violations, Verification

Constraint
A rule limiting acceptable outputs, intermediate states, or transitions. Constraints may be logical, structural, optimization-based, or interface-based.
Constraint Violation
Any output or decision that fails a declared rule or admissibility condition. Benchmarks should distinguish soft violations from hard failures.
Logic Adherence
The extent to which outputs or state transitions satisfy predefined logical rules. It should be measured by explicit tests rather than human impression.
Verification
The act of checking whether an output or state satisfies specified criteria. Verification should be distinguished from generation.
Research Direction

Continuous DPLL and Relaxation

Continuous DPLL
A working term for expressing DPLL-like reasoning operations in differentiable or continuous vector spaces while preserving enough symbolic structure to remain useful.
Differentiable Relaxation
A continuous approximation of a discrete operation that enables gradient-based optimization. Smoothness is not enough; useful structure must remain.
Rule System
An explicit set of symbolic conditions or inference rules used to evaluate or guide model behavior.
Search Procedure
An algorithmic process that explores candidate states, assignments, proofs, or outputs in order to satisfy objectives or constraints.

Optimization and Geometry

Dynamics

Optimization Cognition Terms

Optimization Cognition
A working term for treating cognitive or reasoning dynamics as movement through a high-dimensional objective landscape shaped by prediction error, stability, or consistency.
Prediction Error
The discrepancy between expected and observed state, output, or signal under a given model. The exact definition must be task-aligned rather than abstractly assumed.
Latent Structure
Patterns or relations encoded in hidden variables, learned states, or optimization geometry that are not directly visible from raw outputs alone.
Curvature-Aware Optimization
Optimization that explicitly uses second-order structure, such as Hessians or curvature approximations, to adapt updates or characterize stability.
Second-Order Analysis

Geometry of Learning

Hessian Structure
The organization of second-order partial derivatives of a scalar objective with respect to parameters or states. It captures local curvature and inter-dimensional coupling.
Taylor Expansion
An approximation of a function near a point using derivatives. Second-order Taylor expansions connect local optimization geometry to interpretable updates.
Calibration
The alignment between stated or implied model confidence and empirical correctness. This matters when a control layer aims to improve not just correctness but reliability signals.
Failure Mode
A recurring identifiable way a model or experiment becomes incorrect, unstable, misleading, or unusable. Every major experiment should report them explicitly.

Evaluation and Research Discipline

Measurement

Benchmarks and Metrics

Benchmark
A repeatable evaluation task suite with shared task distribution, metrics, and acceptance criteria.
Contradiction Rate
The proportion of outputs containing mutually incompatible claims. It must always specify whether contradiction is internal, external, or measured against formal rules.
Ablation
A controlled experiment that removes or changes one component to estimate its causal contribution while holding the rest of the system fixed.
Reproducibility
The ability to rerun an experiment and obtain materially consistent results given the same code, configuration, data, and seed policy.
Hypothesis Work

Research Claims

Falsifiable Hypothesis
A claim framed so that possible empirical outcomes could count against it. Improvement claims without rejection conditions are not sufficient.
Hypothesis Tracker
A living document that records claims, rationale, prediction, status, dependencies, and falsification conditions across the research program.
Key Dependency
A prerequisite artifact, benchmark, implementation, or metric that must exist before a hypothesis can be meaningfully tested.
Next Action
The smallest concrete step that materially advances a hypothesis from intuition toward testable execution.

Terms Requiring Formalization

Internal Terms to Handle Carefully

The following expressions are useful inside the lab but should not appear in external papers unless a precise mathematical or empirical definition is provided: continuous DPLL, optimization cognition, cognitive state model, mathematically coherent truth, and deterministic control layer.

Naming

Use Deterministic Architecture for the research stream and control layer for a concrete mechanism or implementation.

Publication

Use Continuous DPLL only when a defined formulation exists, not as a placeholder for intuition about differentiable SAT reasoning.

Revision

Revise the glossary whenever a new benchmark introduces measurement language, a hypothesis depends on an undefined term, or repeated discussion reveals ambiguity.
