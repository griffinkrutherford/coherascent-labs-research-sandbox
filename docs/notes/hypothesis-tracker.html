Lifecycle and Discipline

State Model

Hypothesis States

Draft Active Testing Supported Mixed
  • Draft: promising but not operationalized.
  • Active: concrete enough to drive design.
  • Testing: experiments are underway.
  • Supported or Mixed: assigned only after logged evidence.
Evidence Standard

What Counts as Support

  • The benchmark is explicitly defined.
  • The baseline is fair and relevant.
  • Results reproduce across multiple runs or settings.
  • Failure modes are documented, not hidden.
  • The gain is meaningful relative to cost, latency, or complexity.
Tracker Fields

Required Structure

  • ID and stream to map the claim to the roadmap.
  • Prediction so success is measurable.
  • Variables so experiments are scoped correctly.
  • Falsification so the claim can genuinely fail.
  • Next action so the tracker drives execution.

Deterministic Architecture

H-001

Deterministic control layers reduce contradiction rate in constrained reasoning tasks

Active

Adding a deterministic control layer on top of a probabilistic language model should reduce contradiction rate on structured reasoning tasks relative to unconstrained generation.

Prediction

Contradiction rate drops on logic-constrained benchmarks and the improvement survives multiple seeds or decoding settings.

Variables

Control-layer presence, enforcement type, decoding policy, and task difficulty.

Falsification

No material improvement, gains only on trivial cases, or unacceptable damage to latency or completion quality.

Next Action

Define the benchmark schema and build a first-pass contradiction checker.

H-002

Post-generation verification can recover invalid outputs without full deterministic decoding

Draft

A verifier-guided repair stage may recover a meaningful share of invalid outputs at lower implementation cost than a fully integrated deterministic generation pipeline.

Prediction

Verifier-and-repair should outperform unconstrained generation on logic adherence while remaining simpler than full constrained decoding.

Variables

Verification strategy, repair policy, and allowed retries.

Falsification

Repair rarely succeeds, the verifier is too weak, or total cost approaches stronger methods without similar gains.

Next Action

Define what qualifies as repair rather than regeneration.

Continuous DPLL

H-003

A continuous relaxation of unit propagation can preserve useful logical structure

Active

Some DPLL components, especially propagation-like updates, may be relaxable into a differentiable form that still improves reasoning on clause-based tasks.

Prediction

A propagation-inspired differentiable mechanism should outperform naive continuous baselines on clause consistency or satisfiability-inspired tasks.

Variables

Relaxation design, temperature, training regime, and clause complexity.

Falsification

The relaxation behaves like an unstructured heuristic, gradients are unstable, or simpler methods perform just as well.

Next Action

Formalize a toy setting where unit propagation has a clean continuous analogue.

H-004

Branching is the hardest DPLL component to relax without losing symbolic value

Draft

Branching and backtracking likely resist continuous relaxation more than local propagation because they depend on discrete commitment and failure structure.

Prediction

Local consistency updates should preserve symbolic fidelity more readily than branch-selection analogues on comparable tasks.

Variables

Operation type, relaxation mechanism, and problem size.

Falsification

Branching relaxations work as well or better while retaining interpretable semantics, or the distinction proves experimentally unimportant.

Next Action

Define symbolic fidelity in measurable terms.

Optimization Cognition

H-005

Curvature-aware optimization reveals latent structure that first-order methods obscure

Active

Second-order or curvature-aware analyses may expose stable regimes, couplings, or transition patterns in cognitive-state models that gradient-only summaries fail to reveal.

Prediction

Curvature-aware analysis should explain instability, mode switching, or local minima quality better than first-order summaries alone.

Variables

Optimization method, curvature approximation, objective design, and state-space dimensionality.

Falsification

No additional explanatory value, excessive noise, or complexity without decision-relevant insight.

Next Action

Define a small family of toy objectives with known geometric properties.

H-006

Prediction-error minimization alone is insufficient for reliable reasoning dynamics

Draft

Minimizing prediction error without structural priors or constraints may fit behavior while still producing brittle or uninterpretable internal dynamics.

Prediction

Pure prediction-error objectives should underperform structured variants on stability, interpretability, or perturbation robustness.

Variables

Objective definition, structural priors, and regularization strategy.

Falsification

Pure prediction-error objectives perform on par with structured approaches across robustness and interpretability criteria.

Next Action

Specify candidate priors and a perturbation protocol for comparison.

Neuro-Symbolic Bridge

H-007

Hybrid symbolic-neural systems improve verifiability on reasoning-heavy tasks

Active

A hybrid architecture that combines Transformer representations with explicit symbolic checking or state manipulation should outperform a purely neural baseline on machine-checkable reasoning tasks where verifiability matters.

Prediction

Hybrid systems should achieve higher verified-correctness rates or lower rule-violation rates on reasoning-heavy tasks.

Variables

Architecture type, symbolic interface design, and task family.

Falsification

No verifiability gain under fair controls or symbolic complexity without measurable value.

Next Action

Define the first candidate hybrid interface and a fair baseline.

H-008

Interface design is the bottleneck in neuro-symbolic systems

Draft

The main limitation may not be symbolic capacity itself but the quality of the interface between learned representations and formal structures.

Prediction

Interface changes should explain more performance variance than increasing symbolic rule capacity on matched tasks.

Variables

Interface design, symbolic module complexity, and training or decoding strategy.

Falsification

Increasing symbolic capacity dominates interface changes or interface variants fail to materially alter outcomes.

Next Action

Define a minimal symbolic core that stays fixed while interface variants change.

Cross-Cutting Measurement Questions

MQ-001

How will hallucination be measured?

Early experiments should separate contradiction, unverifiable claim, and failed constraint satisfaction rather than treating hallucination as a single undifferentiated label.

MQ-002

What counts as meaningful improvement?

Small gains may not justify large costs. Each stream should define minimum effect sizes and acceptable latency or complexity overhead before experiments begin.

MQ-003

What level of reproducibility is required?

One successful run is insufficient. The program needs an explicit rerun count, seed policy, and reporting format before any hypothesis is promoted.
